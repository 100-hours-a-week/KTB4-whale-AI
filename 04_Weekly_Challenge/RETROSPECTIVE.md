# 회고록

## Summary

    - 딥러닝 전체 프로세스를 머릿속에 담아둔 상태에서 여러 가지 모델링을 하고 싶었습니다.
    - 기준을 잡고 모델링을 해야 서로 다른 모델의 공통점과 차이점을 명확하게 분석할 수 있기 때문입니다.
    - 우선 딥러닝 전체 프로세스가 소스 코드에서 어떻게 작성되는지 살펴봤습니다.
    - 그리고 개별 한 줄의 코드가 딥러닝 전체 프로세스에서 어떤 의미를 갖는지 정리했습니다.
    - 근본, 현재 트렌드 등의 각 포인트를 연결하는 방식으로 학습을 수행했습니다.
    - 정리하는 과정에서 아래 방향성을 찾을 수 있었습니다.
        1. 전처리에서 중요한 것은 안전한 처리
        2. 증강 모듈의 발전 방향은 실용성
        3. 활성화 모듈의 개선 방향은 문제 해결
    - AI 엔지니어는 전체적인 프로세스가 자연스럽게 머릿 속에 흘러가야 하는 게 기본이라 생각합니다.
    - 내가 작성하는 코드, 내가 사용하는 모듈이 이 구조에서 어떤 의미를 갖는지 하나씩 살펴보는 과정을 이번 위클리 챌린지를 통해 습득할 수 있었습니다.

## Phase 1

1.  numpy array로 변환하는 게 왜 안전한 처리인가?

    ```python
    # 전처리
    y_train = y_train_data.values
    y_val = y_validation_data.values
    y_test = y_test_data.values

    # ...(생략)

    # 모델 학습
    for name, model in models.items():
        # y를 1차원 numpy array로 변환 (안전하게 처리)
        y_train_1d = np.asarray(y_train).ravel()
        y_val_1d   = np.asarray(y_val).ravel()
        y_test_1d  = np.asarray(y_test).ravel()
    ```

    - 우리가 가져온 데이터의 타입을 일관된 형태로 관리하기 위해
      - 여러 가지 데이터 타입 예시
        - pandas.Series
        - numpy.ndarray
        - list
        - pandas.DataFrame의 한 컬럼
    - `np.asarray()`의 역할
      - 일관적인 타입 반환으로 타입 추론이 쉬움
      - 타입이 numpy array이면, numpy array로 반환
      - 타입이 pandas Series이면, numpy array로 반환
    - `.ravel()`의 역할
      - 배열을 1차원으로 펼쳐주는 메서드
      - 대부분의 분류 및 회귀 모델은 1차원을 기대
      - 2차원 배열일 경우, 모델이 제대로 학습되지 않을 수 있기 때문

2. 여러 활성화 함수들 간의 순서(or 관계)
   - 해당 과정을 통해 MLP, CNN을 구현할 때 어떤 활성화 함수를 사용할지 판단하는 데 도움
   - Sigmoid가 근본이고, 나머지는 대부분 기존 활성화 함수가 갖는 문제점을 개선한 형태임
     - $ \sigma(x) = \frac{1}{1 + e^{-x}} $
   - (배경) 실제 생물학적 뉴런의 동작 방식
     - 뉴런은 다른 뉴런으로부터 전기 신호를 받음
     - 입력 신호의 총합이 "일정 수준 이상"이 되면, 전기 신호를 발화
     - 이 발화가 완전한 0 또는 1이 아니라, 입력 강도에 따라 **점진적 발화** 확률이 증가함
     - (소결) 실제 뉴런은 입력의 강도에 따라 반응하는 확률적, 연속적인 특성을 가짐
   - (한계) 초기 인공 뉴런 모델의 경우 계단 함수(Step Function)을 사용
     - 해당 함수는 x=0에서 불연속
     - 생물 뉴런의 연속적인 특성을 제대로 반영하지 못함
     - (소결) Backpropagation(역전파) 알고리즘 사용 불가 (점진적 가중치 수정 불가)
   - (해결) Sigmoid 함수의 등장
     - 역전파의 연쇄 법칙(체인 룰)을 반영하기 위해 미분 가능한 활성화 함수 필요
     - Sigmoid는 해당 문제를 해결하면서, 0 또는 1로만 출력 가능해 생물학적 유사성도 가짐

3. backward가 누적 합산을 default로 선택하는 이유
